## Purpose

Make the tool easier to install, configure, upgrade, and operate repeatedly
after the core profile and quota behavior is stable.

## Goals

- Centralize configuration documentation and defaults.
- Improve install/update verification.
- Add shell-friendly commands for common workflows.
- Polish table rendering for narrow terminals and large profile labels.
- Keep backward compatibility with current commands and profile directories.

## Non-Goals

- Do not introduce a web UI.
- Do not change profile storage layout unless a migration is included.
- Do not add external services.
